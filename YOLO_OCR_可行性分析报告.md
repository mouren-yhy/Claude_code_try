# 微信代管项目 YOLO + OCR 方案可行性分析与实施计划

## 一、背景与问题

### 当前方案问题
1. **坐标模板依赖**：窗口缩放/移动后需重新校准
2. **颜色检测不稳定**：不同主题下阈值需调整，暗黑模式识别率低
3. **无法识别 UI 元素**：无法检测小红点、群聊标识等
4. **OCR 区域不准**：依赖固定坐标导致识别率波动大

### 项目扩展需求
1. **多白名单用户回复**：需要识别当前聊天对象
2. **新消息检测**：通过小红点等方式检测新消息
3. **自动切换聊天**：在多个联系人之间切换并回复

---

## 二、技术选型结论

### 推荐方案：YOLOv8n + PP-OCRv4

| 组件 | 选择 | 理由 |
|------|------|------|
| 目标检测 | YOLOv8n | CPU 80ms、3.2M参数、生态成熟 |
| OCR识别 | PP-OCRv4 | 已集成、中文优化、2M参数 |
| 训练框架 | Ultralytics | API简单、文档完善 |
| 部署格式 | ONNX | 跨平台兼容 |

### 模型对比结论

| 模型 | 精度 | 速度 | 推荐度 |
|------|------|------|--------|
| YOLOv8n | 37.3 mAP | 80ms (CPU) | ⭐⭐⭐⭐⭐ |
| YOLOv9t | 38.3 mAP | 较快 | ⭐⭐⭐⭐ |
| YOLO-NAS | 中等 | 较慢 | ⭐⭐⭐ |
